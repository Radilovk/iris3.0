# Адаптация към текущия код

## Какво беше направено

След сливане на последните промени от главния (`main`) клон, всички функции за поддръжка на множество AI модели бяха успешно адаптирани към текущия код.

## Резултати от адаптацията

### ✅ Запазени функционалности

1. **Multi-provider поддръжка** - Системата продължава да поддържа:
   - Google Gemini (препоръчан по подразбиране)
   - OpenAI GPT-4o и GPT-4o-mini
   - OpenAI-съвместими провайдери

2. **worker.js** - Всички три AI функции са налични и работят:
   - `aiCall()` - рутер между провайдери
   - `aiCallOpenAI()` - за OpenAI API
   - `aiCallGemini()` - за Gemini API

3. **wrangler.toml** - Конфигурацията е актуална:
   ```toml
   AI_PROVIDER = "gemini"
   AI_MODEL = "gemini-2.0-flash-exp"
   GEMINI_API_URL = "https://generativelanguage.googleapis.com/v1beta"
   ```

### 🆕 Нови файлове от main клона

След сливането бяха добавени следните нови файлове:

1. **`.gitignore`** - За игнориране на временни файлове

3. **`start.sh`** - Автоматичен стартов скрипт за Linux/Mac

## Следващи стъпки

За да използвате системата:

### 1. Локално стартиране (Flask app)

**Linux/Mac:**
```bash
./start.sh
```

**Windows:**
```cmd
start.bat
```

### 2. Cloudflare Worker деплой

```bash
# Задайте API ключ
wrangler secret put AI_API_KEY

# Деплой
wrangler deploy
```

### 3. Избор на AI провайдер

#### За Gemini (препоръчан):
1. Вземете API ключ от: https://aistudio.google.com/app/apikey
2. Конфигурацията вече е зададена на `AI_PROVIDER = "gemini"`
3. Безплатен tier е достъпен за тестване

#### За OpenAI (GPT-4o-mini или GPT-4o):
1. Променете в `wrangler.toml`:
   ```toml
   AI_PROVIDER = "openai"
   AI_MODEL = "gpt-4o-mini"  # или "gpt-4o"
   ```
2. Вземете API ключ от: https://platform.openai.com/api-keys
